# Vibe 中介撮合系统 - 用户端详细说明

> 说明：本机器人通过 Telegram 完成订单发布、报名撮合、订单内沟通，并在系统后台对支付/托管状态进行轮询与联动。链上到账以系统确认结果为准；如遇异常可联系管理员协助处理。

## 1. 快速上手（你应该先记住的）
1. 发布需求/服务：`/post`
2. 查看开放订单：`/list`（可筛选需求/服务）
3. 查看订单详情：`/view <id>`
4. 下单沟通：在订单内发消息：`/message <order_id> <text>`
5. 会员后报名接单（服务挂单相关）：`/apply <order_id>`
6. 发布方交付流程：`/applications` → `/choose_assignee` → `/start_work` → `/complete`
7. 发生争议：`/raise_dispute <order_id>`
8. 支付/托管异常：`/restart_payment <order_id>`（或联系管理员）

## 2. 角色与基本权限（你是什么人）
系统里主要用“是否为会员”和“你在订单中的身份”来决定可操作能力。

- 会员（`/profile` 可查看）：通常可以报名“服务挂单”（同一会员未结束挂单会受约束）。
- 发布方：对自己发布的需求/服务挂单拥有编辑、选择接单方、开始执行、完成订单等权限（具体取决于订单当前状态）。
- 接单方：在被选定后参与后续沟通/交付流程。
- 若你在某个操作上提示无权限，通常是订单状态不允许、你不是该订单对应角色、或你不是有效会员。

## 3. 你能做的事（按场景）

### 3.1 查看与沟通
- `/start`  
  初始化并展示欢迎信息与当前会员状态。
- `/help`  
  查看帮助说明。
- `/profile`  
  查看个人资料、会员状态与订单统计。
- `/list [demand|offer]` / `/list_demand` / `/list_offer`  
  分页查看当前开放订单；`demand` 通常是“需求单”，`offer` 通常是“服务挂单”。
- `/view <id>`  
  查看指定订单详情（包含订单主体信息、支付/托管摘要、报名情况、关键状态等）。
- `/my_orders`  
  查看与你相关的订单：你作为发布方或接单方的订单。
- `/message <order_id> <text>`  
  在订单上下文中发送消息（用于沟通交易细节、交付约定等）。  
  若系统使用了子会话语法，你的命令可能会要求类似 `order_id.sub_id` 的格式（以机器人提示为准）。
- `/message_history <order_id> [limit]`  
  查看订单历史聊天（以你权限可见范围为准；默认较多条数时会分页/截断）。

### 3.2 会员相关（成为会员后能接单）
- `/become_member`  
  发起会员费支付流程。支付确认后系统会开通/续期会员，并给出到期提示。
- 会员状态到期后：通常会影响你继续报名接单或进行相关操作。

### 3.3 发布订单（发布方主流程）
入口：`/post`

发布后通常会进入不同状态。你需要根据系统提示走对应流程。

常见链路（按阶段）：
1. `/applications <order_id>`：查看报名池（候选接单方列表）
2. `/choose_assignee <application_id>`：从报名池选择接单方
3. `/start_work <order_id>`：开始执行（通常要求托管金额满足预算等条件；若不足会提示你先追加托管）
4. `/complete <order_id>`：确认订单完成

常见规则提示：
1. 进入“进行中/完成”后，订单部分关键信息将受到限制（系统会按状态保护数据）。
2. 如果系统要求你追加托管才能开始执行，请按提示操作，否则无法进入执行状态。

### 3.4 接单报名（会员侧）
入口：`/apply <order_id>`

- 仅对符合条件的订单开放（例如服务挂单相关）。
- 对同一订单通常不会无限重复创建报名记录；重复报名会返回已有状态/结果。

如果你被选定为接单方：
- 主要后续通过 `/message` 与对方沟通并完成交付。

## 4. 订单变更与异常处理（用户最常遇到）

### 4.1 支付超时/需要重新发起
当你的支付超过系统设定的超时时间，或被判定为失效时，系统会提示你重新操作。

- `/restart_payment <order_id>`  
  用于为草稿/已失效订单重新发起支付流程。  
  系统会提示你“必须严格按金额和收款地址”完成转账，确保后台能正确识别到账。

你需要注意：
1. 严格按系统给出的金额与地址转账（不要少付/多付，地址也必须一致）。
2. 在超时窗口内完成支付，避免再次被标记为失效。

### 4.2 争议与申诉
- `/raise_dispute <order_id>`  
  触发争议流程。之后你需要配合管理员对订单相关的支付/托管/消息记录进行查看与裁决。

### 4.3 取消与删除/恢复
- `/cancel_order <order_id>`  
  发布方取消订单（能否取消取决于订单当前状态，系统会给出明确提示）。
- `/delete_order <order_id>`  
  删除草稿或已失效订单（系统会标记为 `deleted`，不再作为草稿显示）。
- `/restart_payment <order_id>`  
  对草稿/已失效订单重新发起支付流程（与 4.1 类似）。

## 5. 服务挂单专用：买家会话
如果你的订单属于“服务挂单”且进入对应流程，你可以使用：
- `/service_chats <order_id>`  
  用于查看买家会话列表与私聊入口指引（具体以系统提示为准）。

## 6. 常见问题（直接按这个排查）
- “我为什么提示无权限？”
  通常是订单状态不允许、你不是该订单对应角色、或你不是有效会员。
- “支付成功但订单没变？”
  请等待系统后台轮询确认；若后续提示超时/失效，请按提示使用 `/restart_payment <order_id>`。
- “链上对账有困难怎么办？”
  联系管理员协助核对该订单的支付/托管记录。
